# CLAUDE.md — EX04 Project Bible

## What this project is

LangGraph pipeline that reverse-engineers a Python codebase:
1. Parses `.py` files into a knowledge graph (AST → networkx DiGraph)
2. Exports an Obsidian vault (`obsidian/`)
3. Routes through Navigator → Analyzer → Fixer agents via a LangGraph StateGraph
4. **Adaptive**: if the graph has < 5 edges (broken/Python-2 code), takes a `raw_reader` fallback branch instead of `navigate`

**Target codebase**: `data/broken-python` (12 documented bugs across 2 files)  
**Assignment**: EX04, Lecture 07, AI Orchestration Course — Dr. Yoram Segal

## Commands

```bash
# Install
uv sync

# Graph only — no API key needed
uv run python main.py --graph-only

# Full pipeline (needs ANTHROPIC_API_KEY in .env)
uv run python main.py --budget 40000

# Improvement loop
uv run python main.py --improve --iterations 2 --budget 80000

# Print LangGraph Mermaid diagram
uv run python main.py --diagram

# All tests
uv run pytest tests/ -v

# Tests with coverage
uv run pytest tests/ -v --cov=src --cov-report=term-missing

# Lint
uv run ruff check src/ tests/
```

## Architecture

### LangGraph nodes

```
__start__
    └─► build_graph          (parse AST → build KG → export vault)
            ├─► navigate     (dense: edge_count ≥ 5 → NavigatorAgent)
            └─► raw_reader   (sparse: edge_count < 5 → BaseAgent reads raw text)
                    └─► analyze   (AnalyzerAgent → bug_report JSON)
                            └─► fix   (FixerAgent → fix_report JSON)
                                    └─► __end__
```

### SPARSE_EDGE_THRESHOLD = 5
When the parsed graph has fewer than 5 edges (e.g. syntax errors block AST), `is_sparse=True` and the workflow skips `navigate`, uses `raw_reader` instead.

### WorkflowState fields
`source_root`, `vault_dir`, `knowledge_graph`, `graph_summary`, `raw_files`, `navigation`, `bug_report`, `fix_report`, `token_usage`, `is_sparse`, `error`

### Error propagation
Every node checks `state.get("error")` first. If set, returns immediately (no LLM call). Set `state["error"]` to propagate failure through the whole graph.

### Token budget
`AgentBudget(max_tokens)` is shared across all three agents. Raises `TokenBudgetExceededError` when `total_used > max_tokens`. Pass `--budget N` to control.

---

## Agent output format

All agents output **raw JSON, no markdown fences**. Parsers do 3-stage extraction:
1. Strip fences if present
2. `json.loads(text)`
3. Scan for embedded `{...}` if step 2 fails

On failure: `{"parse_error": True, "bugs": []}` or `{"parse_error": True, "fixes": []}` — never silently lost.

## What NOT to do

- Do **not** edit files in `obsidian/` directly — they are regenerated by `ObsidianExporter`
- Do **not** call `anthropic.Anthropic` directly in any node — always go through `BaseAgent._call_llm()`
- Do **not** add code inside JSON values in Fixer prompts — the system prompt explicitly bans this to avoid escaping bugs
- Do **not** skip `AgentBudget.record()` after an LLM call
- Do **not** use `git add .` — the `.env` file has the API key
